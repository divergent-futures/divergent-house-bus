# Comparable Project - Mutiny Motors (@thatdetroitandy)

**Date:** 2026-06-29  ·  A benchmark/kindred open-source-vehicle program to track alongside the House Bus, and a candidate **recurring Divergent Futures theme**.

## What it is
- **Mutiny Motors** by **Andy Didorosi** (YouTube/TikTok/X **@thatdetroitandy**). "The open source affordable truck."
- Vehicle: a **1996 Suzuki Carry kei truck** as the drivetrain-prototype foundation, fitted with a **wrecked Tesla Model 3 Performance drivetrain**.
- **Mission (from the repo):** affordable, repairable, useful, durable, **open source** (build/modify/fork freely), **Detroit-made, by people paid a thriving wage.**
- **License: CERN-OHL-S** (strongly reciprocal open hardware).
- **The engine of it:** he bought and is reviving **an old Detroit car factory**, building with **professionals + volunteers**, posting **daily**, with a real plan to ship.
- Repo: `github.com/ThatDetroitAndy/mutiny` · Channel: `youtube.com/@thatdetroitandy`

## The striking overlaps with House Bus
- **Same thesis:** an open-source, affordable, repairable, fork-able *vehicle* as a public good.
- **Same scavenge play:** a **wrecked Tesla Model 3 Performance drivetrain** - exactly the path we costed in `Bus_V1_Scavenged_Ford_Variant.md`. His build is live validation (and a source to watch for the real CAN/BMS/controller integration - the "tech to pack in").
- **Same license family:** he's on **CERN-OHL-S**; our hardware is CERN-OHL-S too (we add MIT for code, CC-BY for docs).
- **Same values:** repairable, durable, anti-planned-obsolescence, community-forkable.

## The honest contrast - we're near-mirror-images
| | **Mutiny Motors** | **House Bus (us)** |
|---|---|---|
| **Strength** | **Execution + org + community** - a real shop, paid pros, volunteers, daily content, a running drivetrain prototype | **Documentation + engineering rigor** - deep design/sourcing/feasibility/CFD/compliance across every subsystem |
| **Repo state** | **Nascent** - 4 commits, README (mission + license) only; the substance lives in the shop + videos | **Deep** - dozens of design docs, tiered sourcing trackers, models, floor plans, CFD, compliance |
| **Physical build** | **Underway** (factory, team) | **Not started** (design-stage; V1 skoolie + V2 planned) |
| **Vehicle** | tiny kei truck (cheap, high-volume) | 25-27 ft house bus (dwelling) |

So: **he has the build engine and thin docs; we have deep docs and no build engine.** That's the whole lesson.

## What we can learn / potentially adopt (TJ's "is this a model?")
1. **The org model is the template.** Physical shop + **paid professionals + volunteers** + **daily content -> community -> credibility -> funding -> build**. This is exactly the "outreach when the channel has reach" path we flagged for supplier relationships - Andy is *doing* it. Worth studying his sequencing (factory first? content first? funding?).
2. **Drivetrain-prototype-first.** He's proving the scavenged-M3P drivetrain as a standalone module before the whole vehicle - a de-risking pattern we could mirror (build/validate the drive unit + pack integration first).
3. **"Thriving wage / Detroit-made"** as an ethos that draws community + press - a values-forward framing that fits Divergent Futures.
4. **Thin-repo-but-shipping** vs our deep-repo-but-not-building - a nudge that at some point **documentation has to meet a shop.** His model is one way to get there.

## Recurring Divergent Futures theme
Set up a **follow-along series**: track Mutiny's progress alongside the bus as "**two open-source vehicle programs, opposite strengths**" - his execution/org vs our design depth. Angles: the shared M3P drivetrain (compare integration approaches), the shared CERN-OHL licensing, the community/factory model, and honest "what each could learn from the other." Potential for real **cross-pollination or collaboration** (drivetrain, licensing, community playbook) - and a credible reason to actually reach out.

## Actions / open

2. **Watch his M3P integration** - directly informs our scavenged-Ford variant + V2 drivetrain.
3. **Study his org sequencing** (factory / team / funding / content) as a candidate model for the bus program.
4. Consider **reaching out** (shared license, shared scavenge play) once we have channel reach - possible collaboration.
